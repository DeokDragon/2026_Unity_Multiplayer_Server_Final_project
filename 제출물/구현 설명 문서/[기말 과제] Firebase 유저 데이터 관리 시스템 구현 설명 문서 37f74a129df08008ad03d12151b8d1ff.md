# [기말 과제] Firebase 유저 데이터 관리 시스템 구현 설명 문서

작성자 : 김덕용

# 1. 구현한 기능 목록

이번 프로젝트에서는 기본적인 유저 데이터 관리 기능과 더불어, 심화 과제 3종을 모두 추가로 구현하여 완성도를 높였습니다.

- **기본 기능**
    - **회원가입 및 로그인:** 닉네임 입력 기반의 계정 생성 및 중복 방지, 로그인 시 `MainScene` 자동 전환.
    - **코인 및 점수 연동:** `MainUIManager`를 통해 상단 고정 UI에 로그인한 유저의 코인과 최고 점수 실시간 표시.
    - **아이템 상점:** 코인을 소모하여 아이템 구매 시 인벤토리 수량 증가 및 코인 즉시 차감.
    - **인벤토리:** 보유 중인 아이템 수량 표시 및 버튼 클릭 시 아이템 소모(개수 감소) 기능.
- **심화 기능 (전부 구현)**
    - **[심화 1] 유닛 구매 기능:** 상점에서 `unit2`, `unit3`를 코인으로 구매. `UnitList` JSON을 딕셔너리로 역직렬화하여 이미 보유한 유닛은 중복 구매할 수 없도록 예외 처리 구현.
    - **[심화 2] 게임 결과 저장 (최고 점수 갱신):** 게임 종료 시 랜덤 점수(10~500)를 획득하며, firebase에 저장된 기존 점수와 비교. 기존보다 높을 경우 최고 점수를 갱신하고 1000 코인 보상, 갱신 실패 시 50 코인을 보상하는 차등 지급 로직 구현.
    - **[심화 3] 공용 거래소 (Global Auction) 기능:** 모든 유저가 서버의 아이템 수량을 공유하는 거래소 시스템 구축. 판매 시 내 인벤토리는 줄고 서버 수량은 증가하며, 구매 시 서버 수량은 줄고 내 인벤토리가 증가하는 다중 경로 업데이트구현.

# 5. PlayerPrefs에 저장한 값 설명

로그인 및 회원가입 성공 시, 로컬 기기의 `PlayerPrefs`에 아래 두 가지 값을 캐싱하여 다음 씬(`MainScene`)에서 Firebase 데이터베이스에 접근할 때 식별자로 사용했습니다

- **`UserKey`**: Firebase의 `Push().Key`로 생성된 유저의 고유 식별 해시값. (데이터베이스 경로 탐색용)
- **`UserNickName`**: 유저가 입력한 닉네임 문자열.

# 6. Inventory&UnitList JSON 처리 방식 설명

- Firebase에는 복잡한 구조를 쉽게 저장하기 위해 `UnitList`와 `Inventory`를 JSON 형태의 문자열(String)로 변환하여 저장했습니다.
- 데이터를 불러올 때는 `Newtonsoft.Json`의 `JsonConvert.DeserializeObject<Dictionary<string, int>>`를 사용하여 JSON 문자열을 C#의 딕셔너리로 **역직렬화** 하였습니다. 이를 통해 `inventory["IceCrystal"]`과 같이 키 값으로 쉽게 수량을 조회하고 연산할 수 있었습니다.
- 데이터를 수정(구매/판매/사용)한 후에는 다시 `JsonConvert.SerializeObject`를 통해 딕셔너리를 JSON 문자열로 **직렬화**한 뒤 Firebase에 업데이트하였습니다.

# 7. 실행 중 발생한 문제와 해결 방법

1. 데이터 키 값 오타 및 대소문자 충돌 문제
    - **문제:** `UnitList`에서 `Unit2`, `unit2`, `unsit2` 등 대소문자와 오타가 혼용되어, 유닛을 구매해도 보유 상태가 갱신되지 않고 파이어베이스에 새로운 유닛 데이터가 중복으로 생성되는 문제가 있었습니다.
    - **해결:** Firebase는 Key 값의 대소문자를 엄격하게 구분한다는 점을 파악했습니다. `UserData.cs`의 초기 생성부, `ShopManager`의 구매부, `InventoryManager`의 호출부 모두 **`unit2`, `unit3`** 형태의 소문자로 네이밍 컨벤션을 완벽히 통일하고 기존 꼬인 DB를 초기화하여 해결했습니다.
2. UI 씬 통합 시 참조 오류
    - **문제:** 로그인 씬에서 메인 씬으로 넘어온 후 버튼을 눌렀을 때 `inventoryPanel`이 켜지지 않고 에러가 발생했습니다.
    - **해결:** 스크립트 상에서 패널을 열고 닫는 코드를 작성했지만, 유니티 에디터 인스펙터창에서 해당 스크립트 변수에 실제 UI GameObject(패널)를 드래그하여 할당(Assign)하지 않아서 발생한 오류임을 알게 되었습니다. 하이어라키의 각 패널과 매니저를 인스펙터에 정확히 연결하여 해결했습니다.
3. UI 패널 오픈 시 데이터 비동기화 및 갱신 오류
    - **문제:** 상점에서 아이템을 구매한 뒤 인벤토리를 열거나, 다른 유저가 거래소 물품을 구매한 뒤 내가 거래소를 열었을 때, 과거에 불러온 데이터가 화면에 그대로 남아있어 실제 서버(Firebase) 정보와 불일치하는 논리적 오류가 발생했습니다.
    - **해결:** 각 UI 패널을 여는 버튼 이벤트가 실행될 때마다, 단순히 UI만 활성화하는 것이 아니라 해당 매니저의 데이터 호출 함수(`LoadInventory()`, `LoadAuctionData()`)를 강제로 먼저 실행하도록 코드를 추가했습니다. 이를 통해 유저가 패널에 진입할 때마다 Firebase에서 최신 정보를 새로고침하여 가져오도록 동기화 문제를 완벽하게 해결했습니다.
